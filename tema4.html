Fundamentos de Desarrollo Web en Java

Repaso completo de conceptos y tecnologías clave para aplicaciones web en Java

1. Fundamentos del Desarrollo Web en Java

1.1 ¿Qué es Jakarta EE?

Jakarta EE (antes Java EE) es una plataforma de Java para construir aplicaciones empresariales web robustas y escalables. Gestionada por Eclipse Foundation, incluye APIs como JPA, JTA, JAX-RS, CDI, JSF, JSP y Messaging, con Jakarta EE 10 como versión vigente en 2025.

1.2 Servlets

Un servlet es un componente Java que procesa peticiones HTTP (GET, POST, PUT, DELETE) y genera respuestas dinámicas. Su ciclo de vida consta de init(), service() y destroy(). Sigue siendo la base de muchos frameworks como Spring Boot, aunque su uso directo ha disminuido.

1.3 Jakarta Server Pages (JSP)

JSP permite combinar HTML con código Java (<% %>) y compilarlo en servlets. Facilita separar lógica y presentación con JavaBeans y JSTL, y aunque menos popular frente a frameworks modernos, mantiene relevancia en sistemas legacy.

1.4 Tipos de Peticiones HTTP

GET (obtener recursos), POST (crear/actualizar), PUT (actualizar completo), DELETE (eliminar), PATCH (modificar parcialmente), HEAD (solo encabezados) y OPTIONS (opciones de comunicación). Frameworks como Spring manejan estas peticiones.

1.5 JavaBeans

Clases Java con propiedades privadas, getters/setters y constructor sin argumentos, usadas para transferir datos entre capas (por ejemplo, entre servlets y JSP).

1.6 Filtros

Componentes que interceptan peticiones y respuestas para tareas como autenticación, logging o compresión. Se configuran en web.xml o con anotaciones.

1.7 Listeners

Clases que reaccionan a eventos del ciclo de vida web (inicio/cierre de app, sesiones). Por ejemplo, ServletContextListener para inicializar recursos.

2. Arquitectura de Aplicaciones Web en Java

2.1 MVC

Patrón que separa Modelo (datos y lógica), Vista (JSP/HTML/frontends) y Controlador (servlets/Spring MVC). Mejora mantenibilidad y testabilidad.

2.2 Capas

Presentación (UI), Negocio (servicios), Persistencia (bases de datos/API), Integración (servicios externos).

3. Tecnologías y Frameworks Clave

3.1 Spring Framework

Inyección de dependencias, anotaciones, Spring MVC, Spring Boot (autoconfiguración, starters, servidor embebido, actuator). Spring Boot 3.x domina en 2025 con soporte reactivo y GraalVM.

3.2 Jakarta Persistence API (JPA)

ORM estándar con entidades (@Entity, @Table, @Id) y EntityManager para CRUD. Hibernate es la implementación habitual.

3.3 JAX-RS

Especificación REST con @Path, @GET, @POST. Jersey y RESTEasy son implementaciones populares. Ideal para APIs consumidas por frontends o microservicios.

3.4 Jakarta Server Faces (JSF)

Framework de componentes servidor-rendered, menos usado que React/Angular, pero vigente en aplicaciones empresariales.

3.5 Thymeleaf

Motor de plantillas con HTML puro, recomendado en Spring Boot para vistas dinámicas sin la complejidad de JSP.

4. Programación Reactiva

4.1 Programación Reactiva

Paradigma asíncrono y no bloqueante basado en Reactive Streams: asincronía, backpressure, flujos (Flux/Mono).

4.2 Project Reactor

Biblioteca de Reactor: Mono (0-1), Flux (0-N), operadores (map, filter, merge).

4.3 Spring WebFlux

Módulo reactivo de Spring: controladores que retornan Mono/Flux, cliente WebClient. Ideal para alta concurrencia y streaming.

5. Microservicios

5.1 ¿Qué son los Microservicios?

Arquitectura de servicios independientes comunicándose por APIs (REST/gRPC). Ventajas: escalabilidad, flexibilidad y despliegue independiente; desafíos: comunicación, datos distribuidos y monitoreo.

5.2 Spring Cloud

Suite para microservicios: Config, Gateway, Netflix (Eureka, Hystrix), Stream, Sleuth. Facilita descubrimiento, tolerancia a fallos y trazabilidad.

5.3 Patrones de Microservicios

API Gateway, Service Discovery, Circuit Breaker, Distributed Tracing, Event-Driven Architecture con Kafka/RabbitMQ.

6. Otros Conceptos Relevantes

6.1 REST vs SOAP

REST: ligero, HTTP/JSON/XML y stateless. SOAP: basado en XML, con transacciones complejas. SOAP menos común en 2025.

6.2 WebSockets

Comunicación bidireccional en tiempo real con Jakarta WebSocket o Spring WebFlux.

6.3 Seguridad

Spring Security (OAuth2, JWT) y Jakarta Security para autenticación/autorization.

6.4 Contenedores y Orquestación

Docker para empaquetado; Kubernetes para orquestación de microservicios.

6.5 GraalVM

Ejecuta Java nativo para mayor rendimiento y menor memoria. Spring Boot 3.x soporta compilación nativa.

6.6 Bases de Datos

Relacionales (MySQL, PostgreSQL via JPA/Hibernate), NoSQL (MongoDB, Cassandra con Spring Data), Reactivas (R2DBC).

6.7 Pruebas

JUnit para unitarias, Mockito para mocks, Testcontainers para integración con contenedores.

7. Tendencias en 2025

  • Programación reactiva para alta concurrencia.
  • Microservicios con Spring Cloud y Kubernetes.
  • Compilación nativa con GraalVM.
  • Serverless en AWS Lambda y Azure Functions.
  • APIs GraphQL como alternativa a REST.
