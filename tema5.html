¿Qué es la programación reactiva?

La programación reactiva es una forma de programar que se enfoca en reaccionar a los datos cuando llegan, en vez de esperar a que todo esté listo antes de empezar.

Comparación sencilla

Programación tradicional:
👉 Imagina que estás cocinando y no haces nada más hasta que la comida esté lista.
Así funciona la programación tradicional: espera que una tarea termine para pasar a la siguiente.

Programación reactiva:
👉 En cambio, con programación reactiva, puedes hacer otras cosas mientras la comida se cocina.
Cuando esté lista, te avisa (reacciona), y tú actúas.

¿Para qué sirve?

  • Para manejar muchas peticiones al mismo tiempo sin bloquear el sistema
  • Para aplicaciones web más rápidas y escalables
  • Muy útil cuando trabajas con APIs, bases de datos, streaming, sensores, etc.

Palabras clave que vas a ver

  • Mono<T>: representa una sola respuesta (como una promesa)
  • Flux<T>: representa muchas respuestas (una secuencia de datos)
  • subscribe(): te suscribes a los datos cuando estén listos
  • non-blocking: significa que no bloquea el programa mientras espera

¿Qué es el pom.xml?

El archivo pom.xml es como el "manual de instrucciones" de tu proyecto Java. La palabra POM significa Project Object Model.

📦 Es un archivo en formato XML que define todo lo necesario para construir tu proyecto, como:

  • Dependencias (librerías que necesitas)
  • Versión de Java
  • Nombre del proyecto
  • Plugins para compilar, testear o empaquetar
  • Configuración de versiones de Spring, R2DBC, etc.

¿Para qué sirve?

  1. 📚 Agregar dependencias
    ¿Necesitas usar Web, JPA, WebFlux, MySQL o cualquier librería? Solo lo defines así: 
    

    org.springframework.boot
    spring-boot-starter-web
    Maven descarga automáticamente las versiones correctas desde internet.
  2. 🛠 Controlar cómo se construye tu aplicación
    Define cosas como:
    · Qué plugins usar para compilar (maven-compiler-plugin)
    · Qué versión de Java usar (<java.version>)
    · Cómo generar un .jar o .war
  3. 📦 Empaquetar la app en un solo archivo ejecutable
    Con Maven puedes correr un comando: 
    
mvn clean install
    Y Maven crea un archivo .jar con todo lo necesario para ejecutar tu app.

🧪 ¿Y si usas Spring Boot?

Spring Boot se basa en Maven y el pom.xml para saber:
· Qué versiones usar de sus módulos
· Cómo integrar WebFlux, JPA, Seguridad, etc.
· Cómo compilar correctamente según tu JDK

application.properties vs application.yml

📘 application.properties

· Formato de texto plano
· Cada propiedad va en una línea
· Ejemplo: 


server.port=8080
spring.datasource.url=jdbc:mysql://localhost:3306/db

📘 application.yml

· Formato YAML (estructurado)
· Permite anidación clara y ordenada
· Ejemplo: 


server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/db

🤔 ¿Cuál es mejor?

Característica .properties .yml
Más legible ❌ Menos ✅ Más clara y organizada
Soporta estructuras ❌ No ✅ Sí (anidaciones)
Más común en Spring ✅ Sí ✅ Sí
Menos propenso a error ✅ Sí ❌ YAML es sensible a espacios

📄 service/PersonService.java


package com.arle.demoreactiveapi.service;

import com.arle.demoreactiveapi.model.Person;
import com.arle.demoreactiveapi.repository.PersonRepository;
import org.springframework.stereotype.Service;
import reactor.core.publisher.Flux;
import reactor.core.publisher.Mono;

@Service
public class PersonService {

    private final PersonRepository repository;

    public PersonService(PersonRepository repository) {
        this.repository = repository;
    }

    public Mono create(Person person) {
        return repository.save(person);
    }

    public Flux getAll() {
        return repository.findAll();
    }

    public Mono getById(Long id) {
        return repository.findById(id);
    }

    public Mono update(Long id, Person newData) {
        return repository.findById(id)
                .flatMap(existing -> repository.save(new Person(id, newData.name(), newData.email())));
    }

    public Mono delete(Long id) {
        return repository.deleteById(id);
    }
}

📄 controller/PersonController.java


package com.arle.demoreactiveapi.controller;

import com.arle.demoreactiveapi.model.Person;
import com.arle.demoreactiveapi.service.PersonService;
import org.springframework.web.bind.annotation.*;
import reactor.core.publisher.Flux;
import reactor.core.publisher.Mono;

@RestController
@RequestMapping("/api/persons")
public class PersonController {

    private final PersonService service;

    public PersonController(PersonService service) {
        this.service = service;
    }

    @PostMapping
    public Mono create(@RequestBody Person person) {
        return service.create(person);
    }

    @GetMapping
    public Flux getAll() {
        return service.getAll();
    }

    @GetMapping("/{id}")
    public Mono getById(@PathVariable Long id) {
        return service.getById(id);
    }

    @PutMapping("/{id}")
    public Mono update(@PathVariable Long id, @RequestBody Person person) {
        return service.update(id, person);
    }

    @DeleteMapping("/{id}")
    public Mono delete(@PathVariable Long id) {
        return service.delete(id);
    }
}

Tarea: Crear 2 APIs REST con Spring Boot y Programación Reactiva

Crea dos APIs REST reactivas usando Spring Boot, Spring WebFlux, Spring Data R2DBC, y PostgreSQL como base de datos. A continuación, se describen las especificaciones para cada API:

1. API de Libros

Crea una API para gestionar libros con las siguientes características:

  • Entidad: Book (id, title, author, publicationYear)
  • Operaciones CRUD:
    • POST /api/books: Crear un libro
    • GET /api/books: Obtener todos los libros
    • GET /api/books/{id}: Obtener un libro por ID
    • PUT /api/books/{id}: Actualizar un libro
    • DELETE /api/books/{id}: Eliminar un libro
  • Base de datos: PostgreSQL
  • Estructura: Similar a la del ejemplo proporcionado (model, repository, service, controller)

Script de base de datos (PostgreSQL)


CREATE DATABASE books_db;

\c books_db;

CREATE TABLE book (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    author VARCHAR(100) NOT NULL,
    publication_year INTEGER NOT NULL
);

Dependencias en pom.xml

Asegúrate de incluir las siguientes dependencias en el pom.xml: 



    org.springframework.boot
    spring-boot-starter-webflux


    org.springframework.boot
    spring-boot-starter-data-r2dbc


    org.springframework.boot
    spring-boot-starter-validation


    io.r2dbc
    r2dbc-postgresql


    org.postgresql
    postgresql
    runtime

Configuración en application.yml


spring:
  r2dbc:
    url: r2dbc:postgresql://localhost:5432/books_db
    username: postgres
    password: your_password
